<task type="auto">
  <name>Task 1: Set Dictus identity in tauri.conf.json</name>
  <files>src-tauri/tauri.conf.json</files>
  <read_first>
    - src-tauri/tauri.conf.json
    - .planning/phases/01-bundle-identity/01-CONTEXT.md
  </read_first>
  <action>
Edit src-tauri/tauri.conf.json to make these exact changes:

1. **BNDL-01** — Change `"productName": "Handy"` to `"productName": "Dictus"`
2. **BNDL-02** — Change `"identifier": "com.pais.handy"` to `"identifier": "com.dictus.desktop"`
3. **Version** — Change `"version": "0.8.2"` to `"version": "0.1.0"`
4. **BNDL-04** — Change `"createUpdaterArtifacts": true` to `"createUpdaterArtifacts": false`
5. **BNDL-04** — Replace the entire `"plugins.updater"` object (which currently contains `pubkey` and `endpoints` keys) with an empty object: `"updater": {}`
6. **BNDL-05** — Remove the `"signCommand"` field entirely from `"bundle.windows"`. The `"windows"` object should only contain the `"nsis"` key after this edit.

DO NOT modify any other fields. Keep `$schema`, `build`, `app`, `bundle.active`, `bundle.targets`, `bundle.resources`, `bundle.license`, `bundle.icon`, `bundle.macOS`, `bundle.linux`, and `bundle.windows.nsis` exactly as they are.

IMPORTANT: `createUpdaterArtifacts` must be the boolean `false`, not the string `"false"`.

Known dev-mode behavior (not a bug): After this change, `tauri dev` will still show "handy" in the macOS dock because dev mode uses the Cargo binary name. Release builds (`tauri build`) will correctly show "Dictus". This is expected — the binary name rename is deferred to V2 (TECH-03).
</action>
