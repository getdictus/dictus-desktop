# Phase 07 Plan 01: macOS Clean Shutdown Summary

**Quitting Dictus on macOS (tray → Quit Dictus, and post-auto-update relaunch) no longer triggers the OS "Dictus quit unexpectedly" crash dialog. Root cause diagnosed and committed before the fix per SHUT-01 gate; fix applies graceful cleanup at both exit sites in lib.rs; debug-only simulate_updater_restart trigger added for SHUT-03 validation without cutting a throwaway release; UPSTREAM.md extended to protect the divergence from future upstream syncs.**

## Decisions Made

**1. Path (a) graceful cleanup vs path (b) `std::process::exit(0)`**
The plan left both paths open pending the `.ips` diagnosis. The crash report pointed at `tauri-plugin-global-shortcut` Drop releasing CGEventTap from `__cxa_finalize_ranges` — i.e. the C-runtime atexit pass, *after* the Tauri runloop had torn down. Path (a) fixes this by releasing CGEventTap *before* `app.exit`, while the runloop is still alive to process the release. Path (b) was intentionally NOT introduced; it is reserved for a hypothetical Plan 7-02 if (a) had not cleared the dialog.

**2. UI-level debug gating rather than `#[cfg(debug_assertions)]`**
The `simulate_updater_restart` command ships in release binaries. The UI trigger (`SimulateUpdaterRestart.tsx`) is rendered conditionally on `settings.debug_mode`. This lets Pierre validate the updater-relaunch path from a production install without cutting a throwaway test release, at the cost of a small attack surface (a user toggling debug mode could trigger a restart — acceptable, same as other debug-panel commands).

**3. UPSTREAM.md conflict-rule row instead of a verify-sync.sh assertion**
The `flush_and_exit` pattern varies in exact location (currently ~line 255 and ~line 623 in `lib.rs`, but upstream Handy may refactor). A grep-based assertion would be too shape-sensitive and either produce false positives or miss divergences. The human-reviewer table entry is the contract instead — this mirrors the existing protection pattern for `tauri.conf.json`, `llm_client.rs`, and the i18n locales.

**4. Closing the phase without reproducing the crash a second time**
Pierre's memory flagged the macOS quit crash as potentially non-reproducible under clean env (multi-install pollution as a possible contributor). After the SHUT-02 fix landed, Pierre ran daily builds for several days — no dialog appeared. The fix is defensible on first principles (flushing logs + releasing CGEventTap on a live runloop is correct regardless of whether the atexit-path was the *only* trigger), and the validation window is sufficient to close the phase.
