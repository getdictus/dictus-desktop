### Critical constraints (do not violate)
- `createUpdaterArtifacts` MUST be in the `bundle` section, NOT inside `plugins.updater`. Wrong section = no .sig files generated (Pitfall A3 in 04-RESEARCH.md).
- `pubkey` MUST be the raw base64 string exactly as it appears in `~/.tauri/dictus.key.pub`. Do NOT include `-----BEGIN PUBLIC KEY-----` / `-----END PUBLIC KEY-----` wrappers. Do NOT base64-decode and re-encode. Do NOT trim/strip whitespace inside the string (there is none in `tauri signer generate` output). PEM-wrapping causes runtime `UnexpectedKeyId` errors.
- `endpoints` MUST use the DIRECT ASSET DOWNLOAD URL (`.../releases/latest/download/latest.json`), NOT the GitHub Releases HTML page URL (`.../releases/latest`).
- Do NOT modify ANY other field in tauri.conf.json (productName, version, identifier, bundle.targets, bundle.icon, bundle.macOS, bundle.linux, bundle.windows, app.*, build.*, $schema). Leave all unrelated fields exactly as-is.
- Do NOT reorder keys. Tauri's config schema tolerates any order but diff-noise hides the actual change.
