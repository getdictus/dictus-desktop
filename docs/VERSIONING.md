# Versioning

Dictus Desktop uses two parallel version/tag systems that must not be confused.

## App version (semver, public)

**Single source of truth:** `src-tauri/tauri.conf.json` → `"version"`.

- Format: `0.x.y` (semver, zero-major while the product is early-stage)
- Mirrored in `package.json` → `"version"` (keep in sync)
- Published as GitHub Releases on `getdictus/dictus-desktop`
- Consumed by the tauri-updater endpoint (`latest.json`)

**Git tags for app versions:** `v0.x.y` — one per published release.

- Tag is created by the release workflow (`release.yml` / `build.yml`)
- Tag points at the commit that produced the release artifacts
- Pushed to remote; immutable once published

Current app version: **0.1.0** (first public Dictus release, 2026-04-11).

## GSD milestones (internal, planning-only)

Milestones (`v1.0`, `v1.1`, `v1.2`, ...) are **internal planning units** used by
the `get-shit-done` workflow. They group phases and requirements.

- Live in `.planning/MILESTONES.md` and `.planning/milestones/v{X.Y}-*.md`
- **Do NOT create git tags for GSD milestones** — the `v1.x` namespace would
  collide with the `v0.x.y` semver space and pollute `git tag -l`
- A single GSD milestone may span zero, one, or several app releases

**Why separate:** milestones are about planning scope and internal progress;
app versions are about what users actually receive. A milestone like
"v1.2 Performance" could ship across `v0.1.5`, `v0.1.6`, `v0.2.0` — there is
no one-to-one mapping.

### Workflow opt-out

`/gsd:complete-milestone` defaults to creating a git tag. In this repo we skip
that step. If a milestone is archived and a `v1.x` tag was created by mistake,
delete it locally before pushing:

```bash
git tag -d v1.x   # local only
```

## Release checklist (app version bump)

1. Bump `src-tauri/tauri.conf.json` → `version`
2. Bump `package.json` → `version` (keep aligned)
3. Commit: `chore(release): bump version to 0.x.y`
4. Trigger `release.yml` via `workflow_dispatch` → tag `v0.x.y` is created and
   assets published; `latest.json` is generated by `tauri-action`
5. Verify: `curl -sL https://github.com/getdictus/dictus-desktop/releases/latest/download/latest.json | jq`

## Legacy tags

The git history contains tags inherited from the Handy upstream (`v0.7.x`,
`v0.8.x`). They are kept for reference but are **not** Dictus app versions.

- `v0.1.0` (2026-04-11) — first actual Dictus release published by Plan 04-04
- `v0.1.0-dictus` (2026-04-10) — legacy marker from v1.0 GSD milestone
  completion; predates the real release. Candidate for deletion.
